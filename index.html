Manual testing VS Automation testing

Manual testing is performed by a human sitting in front of a computer carefully executing the test steps.

Automation testing means using an automation tool to execute your test case suite.

Goal of automation is to
reduce number of test cases to be run manualy and not eliminate manual testing all together.

What is AT used for?

  1. Time and cost consuming
  2. Multilingual sites manually
  3. You can run automated test unattended (overnight)
  4. Speed of test execution & Test Coverage
  5. Error prone

Which Test Cases to Automate?

Using the following criterion to increase the automation ROI

  • Test cases that are repeatedly executed
  • Test Cases that are very tedious or difficult to perform manually
  • Test Cases which are time-consuming
  • High Risk – Business Critical test cases

The following category of test cases are not suitable for automation:

  • Test Cases that are newly designed and not executed manually at least once
  • Test Cases for which the requirements are frequently changing
  • Test cases which are executed on an ad-hoc basis.

How to Choose an Automation Tool?

Following criterion will help you select the best tool for your requirement.

  • Ease of use
  • Support for various types of test – including functional, test management, mobile, etc…
  • Support for multiple testing frameworks

Tool selection

First, Identify the requirements, explore various tools and its capabilities

Set the expectation from the tool and go for a Proof Of Concept.

Framework for Automation

A framework is set of automation guidelines which help in:

  • Maintaining consistency of Testing
  • Less Maintenance of code
  • Improve re-usability
  1. Data Driven Automation Framework
  2. Keyword Driven Automation Framework
  3. Modular Automation Framework
  4. Hybrid Automation Framework
